### 2. Login and Navigation (AUTH integration)
- Open http://localhost:5173 (or frontend dev URL)
- Login with test account (student role)
- From Dashboard, click on a project card
- Verify: Navigation to /projects/:id works
- Verify: ProjectDetail page loads with project info

### 3. Document Generation (DOC-01~04)
- On ProjectDetail page, click "生成PRD" button
- Verify: Loading overlay appears
- Verify: Generation completes (may take 5-30s)
- Verify: PRD document content appears in editor
- Verify: Content has proper structure (项目概述, 功能需求, etc.)
- Repeat for Frontend and Backend documents
- Test "一键生成全部" button

### 4. Domain Differentiation (D-08/09)
- Create project from SE domain topic
- Generate PRD: Verify content includes functional flow sections
- Create project from BD domain topic
- Generate PRD: Verify content includes data pipeline sections

### 5. Document Editing (DOC-05)
- Edit PRD content in MarkdownEditor
- Type content changes
- Wait 500ms+ after last keystroke
- Verify: "保存中..." indicator briefly appears
- Verify: Changes persist after page refresh

### 6. Tech Stack Management (DOC-06/07)
- View tech stack panel on ProjectDetail
- Verify: Topic techStack displayed as tags
- Click edit: Add/remove tech stack items
- Verify: Changes reflected in display

### 7. Tab Navigation
- Click between PRD/Frontend/Backend tabs
- Verify: Correct document content shows
- Verify: "未生成" indicator for empty documents

### 8. Error Handling
- Try generating for other user's project (should fail with 403/404)
- Test timeout scenario (if MiniMax slow)
- Verify: Proper error messages shown

### 9. Cross-Session Persistence
- Generate documents
- Logout and login again
- Navigate to same project
- Verify: Documents still present with saved content
