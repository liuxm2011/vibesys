### Component Boundaries

| Component | Responsibility |
|-----------|---------------|
| **认证服务** | SSO集成、JWT生成、会话管理 |
| **选题服务** | 选题CRUD、分类管理、锁定逻辑 |
| **文档服务** | 文档存储、模板管理、版本控制 |
| **AI服务** | API调用、限额管理、Prompt构建 |
| **用户服务** | 用户信息、项目关联 |
| **管理服务** | 后台数据、统计、配置 |

### Data Flow

```
用户登录 → SSO认证 → JWT Token → 后端验证
    ↓
选题 → 选题服务 → 项目记录创建
    ↓
生成文档 → AI服务 → Claude/OpenAI API → 文档内容
    ↓
保存文档 → 文档服务 → MySQL存储
    ↓
导出 → OSS存储 → Markdown文件下载
```

## Build Order (Phase Sequence)

| Phase | Build What | Dependency |
|-------|------------|------------|
| 1 | 认证 + 基础用户 | 无 |
| 2 | 选题管理 + 学生端基础 | Phase 1 |
| 3 | 文档模板 + AI服务 | Phase 2 |
| 4 | 管理后台 | Phase 1, 2, 3 |
| 5 | 导出 + 完善 | Phase 3 |

## Key Design Decisions

1. **前后端分离** — 独立开发部署，便于维护
2. **JWT认证** — 与SSO集成后生成JWT，便于后续请求验证
3. **AI服务独立模块** — 集中管理API调用和限额
4. **Redis缓存** — API限额计数、会话缓存
